Все эти действия происходят только для сотрудников с должностью:
Директор (100 часов за пятилетку), Провизор (100 часов), Фармацевт (40 часов).

0. При добавлении нового пустого сотрудника сумма часов на курсах автоматически = 0,
а даты пятилетки и дедлайна курсов не существуют (смотри пункт 1 и 2).

1. При добавлении образования, если дата окончания УО больше даты начала пятилетки, то
дата начала пятилетки принимает значение даты окончания УО. Дата конца пятилетки + ровно 5 лет.
Дата дедлайна для курсов принимает значение даты конца пятилетки.

2. При добавлении категории, если дата получения больше даты начала пятилетки, то
дата начала пятилетки принимает значение даты получения категории. Дата конца пятилетки + ровно 5 лет.
Дата дедлайна для курсов принимает значение даты конца пятилетки.

3. Если у сотрудника рассчитаны даты (то есть сделан пункт 1 или 2), то программа разрешит
добавить курс. Если дата начала курса больше даты начала актуальной пятилетки,
а дата конца курса меньше даты дедлайна, то обновится сумма часов.
Если курс "старый", то он просто добавится в базу без изменений суммы часов.

4. Если при прохождении пункта 3 выяснится, что по новой рассчитанная
сумма часов >= необходимой суммы для должности данного сотрудника, то
сумма часов = 0 и дата дедлайна для курсов + ровно 5 лет.

5. При каждом запуске программы выполняется функция, которая проверяет даты пятилеток
всех сотрудников. Если дата конца пятилетки < сегоднящней даты, то рассчитываются новые значения.
Новая дата начала пятилетки = старая дата конца пятилетки + 1 день.
Новая дата конца пятилетки = старая дата конца пятилетки + 5 лет и 1 день.
